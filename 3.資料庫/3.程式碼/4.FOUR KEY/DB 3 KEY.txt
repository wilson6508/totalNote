PRIMARY Key(PK) (主鍵)
是一種 index 但不能為空值(NULL)，PK 會自動建立 index

適用情境：用來明確識別在資料當中的每一筆資料，是不同的資料。
每筆資料的主鍵不能重複

主鍵常見的使用型態：
int
GUID/UUID

----------------------------------------------------------------------------

Index (索引鍵)

資料索引，可加快搜尋速度
index建立會占用儲存空間，資料增刪修時會異動

適用情境：設定在常用來當作查詢條件的欄位
目的：藉由資料庫的演算法提高查詢速度
成本：占用額怪的磁碟空間

----------------------------------------------------------------------------
Unique (不重複鍵)

和 index 相同，差異在於不能允許重複值

適用情境：資料不能重複的欄位，可以接受 NULL 值。
例如：家庭成員的手機號碼，可以設定為UNIQUE
